css sprites实现原理

2016-12-20

一直听说雪碧图可以减少http请求数，提升网页性能，却一直不知道怎么实现的，啊，小菜鸟。。。
这两天有空学习了下，赶紧记下来，哦耶

先来说说它的优缺点以及使用情景
1.优点：众所周知，减少http请求数，减少图片数目。
2.缺点：维护比较麻烦，如果页面背景有少许改动，一般就要改这张合并的图片，无需改的地方最好不要动，这样避免改动更多的css，如果在原来的地方放不下，有只能（最好）往下加图片，这样图片的字节就增加了，因为每次的图片改动都得往这个图片删除或添加内容，显得稍微繁琐。不过现在有一些在线的sprites图片合成工具也还好吧，比如GoPng，挺好用的。
由于图片的位置需要固定为某个绝对数值，这就失去了诸如center之类的灵活性。
3.适用场景：小图标，大流量网站，小流量的小型网站还是没有必要花太多力气搞这个的吧。而且，现在字体图标，似乎是个更好的解决方案。

进入正题：
雪碧图的实现原理主要是把小图标通过ps或者雪碧图在线生成工具生成，定位的话可以通过ps的切片工具来实现，也可以自己计算。
background-position计算小图标的位置原理
background-position是以背景图片的左上角为原点的，下面给出几个例子，就很容易理解啦。
说明：摘自background-position 用法详细介绍

  1. background-position:0 0;
    背景图片的左上角将与容器元素的左上角对齐。该设置与background-position:left top;或者background-position:0% 0%;设置的效果是一致的。例如：

    .container{
    width:300px;
    height:150px;
    background:transparent url(bg.jpg) no-repeat scroll 0 0;
    border:5px solid green;
    }

效果如下图：

  1. background-position:-70px -40px;
    图片以容器左上角为参考向左偏移70px，向上偏移 40px，示例：

    .container{
    width:300px;
    height:150px;
    background:transparent url(bg.jpg) no-repeat scroll -70px -40px;
    border:5px solid green;
    }

效果如图:

  1. background-position:70px 40px;
    图片以容器左上角为参考向右偏移70px，向下偏移 40px，示例：

    .container{
    width:300px;
    height:150px;
    background:transparent url(bg.jpg) no-repeat scroll 70px 40px;
    border:5px solid green;
    }

效果如图：

PS获取background-position位置教程

通过ps软件利用切片工具画出要显示图片区域后，双击即可弹出“切片详细卡”，可以得到X（靠左距离值），Y（靠上距离值）即可获得对应图片素材在大图中位置。
